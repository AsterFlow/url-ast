# Change History

Complete change history for url-ast.

---

## [3.0.0] - 2026-03-25

### Added

- Type annotation syntax using the `.` operator in route segments and query params (e.g., `/users/:id.number` and `?page.number`).
- `=` operator as a default value initializer for variables and query params (in conjunction with `.type`, when applicable).
- `~` operator to mark parameters or segments as optional (possibly `undefined`), e.g., `/posts/[~slug]`.
- Support for enumerated types in inference (`enum[...]` / `enum`).
- `src/types/ast.ts` module with AST types and JSON formats (`NodeJSON`, `ASTJSON`, etc.) aligned with more standardized AST conventions.
- Build migrated to **esbuild** (CJS/ESM), targeting **ES2024**, including auxiliary bundle for `decodeURL`.

### Changed

- The AST was **redesigned**: the tree now uses stronger abstractions (node types and internal expressions), instead of mirroring each operator/token in an overly granular manner.
- Grammar delimiters and symbols reorganized: `Delimiters` and `EncodingSymbols` were replaced by `ParameterDelimiters`, `DynamicVariableDelimiters`, `GeneralDelimiters`, `Operators`, and `CatchAllExpression`.
- Internal expressions renamed/restructured (e.g., `Null` → `None`, `Value` → `Type`, new states for dynamic and *catch-all* segments); `ContentTypes.Array` adjusted numerically.
- TypeScript inference for `ParseParams` / `ParseSearch` / `TypeMap` updated to support the new syntax (`.type`, `=default`, `~`, and `enum`).

### Removed

- The old syntax that **only** used `=` for type annotations in route parameters (e.g., `:id=number`); the typing role has migrated to `.`.

### Documentation

- **README.md** rewritten for v3: `.type` / `=` / `~`, routes with `:` and `[]`, catch-all `[...]`, query and fragment behavior with/without base, `Analyze.create` / `fromBuffer` / `getBuffer`, AST `display` / `toJSON`, public exports table, Node 20 + TypeScript ^5 requirements.
