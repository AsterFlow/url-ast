# Extracting Protocol, Host, and Port

**url-ast** is not limited to the path and query string — it also performs full parsing of the URL's **origin**, including protocol, hostname, and port.

## Available methods

| Method | Return | Example |
| :--- | :--- | :--- |
| `getProtocol()` | `string \| undefined` | `'https'` |
| `getHostname()` | `string \| undefined` | `'api.example.com'` |
| `getPort()` | `string \| undefined` | `'8080'` |
| `getPathname()` | `string` | `'/users/42'` |

## Full example

<LiveCode initialCode={`import { Analyze } from 'url-ast'

const parsed = new Analyze('https://api.example.com:8080/users/42?active=true#profile')

console.log(parsed.getProtocol())  // 'https'
console.log(parsed.getHostname())  // 'api.example.com'
console.log(parsed.getPort())      // '8080'
console.log(parsed.getPathname())  // '/users/42'`} />

## Combining with template

When used with a template, the origin from the concrete URL is extracted while the parameters are parsed against the template:

<LiveCode initialCode={`const template = new Analyze('/api/users/:id.number')
const parsed = new Analyze('https://myapp.com/api/users/99', template)

parsed.getProtocol()  // 'https'
parsed.getHostname()  // 'myapp.com'
parsed.getParams()    // { id: 99 }`} />

## AST nodes

Internally, the AST represents the origin in three dedicated nodes. You can access them directly:

<LiveCode initialCode={`const parsed = new Analyze('https://example.com:443/path')
const origin = parsed.ast.getOriginNodes()

origin.protocol  // Node { state: OriginExpression.Protocol, ... }
origin.hostname  // Node { state: OriginExpression.Hostname, ... }
origin.port      // Node { state: OriginExpression.Port, ... }`} />

<Callout type="warning">
  If the URL does not contain an origin (e.g., `/users/42`), the `getProtocol()`, `getHostname()`, and `getPort()` methods return `undefined`.
</Callout>
