# Parser and AST Architecture

This page explains the internal architecture of the parser and the structure of the abstract syntax tree (AST) generated by url-ast.

## Scannerless parser

url-ast uses a **scannerless** parser: there is no separate lexer. Tokenization and tree construction happen in a **single pass** over the input string.

```
Input: "/users/:id.number?page.number"
         ↓
   ┌─────────────────┐
   │  AST.parser()   │  ← Char-by-char reading
   │  (scannerless)  │     State identification
   │                 │     Node construction
   └────────┬────────┘
            ↓
   Node[] tree
```

The parser identifies special characters (delimiters, operators) by their **charCode** and transitions between internal states (`InternalExpression`).

## Node structure

Each `Node` in the tree contains:

| Field | Type | Description |
| :--- | :--- | :--- |
| `state` | `AllValues` | Node type/state (numeric enum) |
| `start` | `number` | Start offset in the original string |
| `end` | `number` | End offset in the original string |
| `body` | `Node[]` | Child nodes (sub-tree) |

## Semantic layers

The AST organizes nodes into semantic layers that reflect URL structure:

```
URL: "https://api.example.com:8080/users/:id.number?page=2#section"

┌─────────────────────────────────────────────────────┐
│ Origin                                              │
│  ├─ Protocol  → "https"                             │
│  ├─ Hostname  → "api.example.com"                   │
│  └─ Port      → "8080"                              │
├─────────────────────────────────────────────────────┤
│ Path                                                 │
│  ├─ PathSegment "users"          (static)            │
│  └─ DynamicSegment ":id"                             │
│      ├─ VariableDeclaration "id"                     │
│      └─ TypeAnnotation "number"                      │
├─────────────────────────────────────────────────────┤
│ Query                                                │
│  └─ ParameterDeclaration "page"                      │
├─────────────────────────────────────────────────────┤
│ Fragment                                             │
│  └─ FragmentIdentifier "section"                     │
└─────────────────────────────────────────────────────┘
```

You can access each layer directly:

```typescript
const ast = parsed.ast

ast.getOriginNodes()   // { protocol?, hostname?, port? }
ast.getPathNodes()     // Node[] of route segments
ast.getQueryNode()     // Node of the ? delimiter
ast.getFragmentNode()  // Node of the # delimiter
```

## Internal states

The parser uses numeric enums to identify each node type:

### `InternalExpression`

| Name | Value | Meaning |
| :--- | :--- | :--- |
| `None` | 0 | Untyped node (literal content) |
| `Dynamic` | 200 | Dynamic segment (`/:param` or `[param]`) |
| `DynamicCatchAll` | 201 | Catch-all (`[...slug]`) |
| `DynamicOptionalCatchAll` | 202 | Optional catch-all (`[[...slug]]`) |
| `Default` | 203 | Default value |
| `Wildcard` | 250 | Wildcard expression (`*`) |
| `Path` | 251 | Path segment |
| `Variable` | 252 | Variable declaration |
| `Fragment` | 253 | Fragment identifier |
| `Parameter` | 254 | Parameter declaration |
| `Type` | 255 | Type annotation |

### `ContentTypes`

| Name | Value | JavaScript type |
| :--- | :--- | :--- |
| `Array` | 239 | `string[]` |
| `Enum` | 240 | String union |
| `Boolean` | 247 | `boolean` |
| `String` | 248 | `string` |
| `Number` | 249 | `number` |

## Semantic tokens

The `SemanticTokens` map translates numeric codes to readable names:

```typescript
import { SemanticTokens } from 'url-ast'

SemanticTokens[251] // 'PathSegment'
SemanticTokens[249] // 'NumberType'
SemanticTokens[200] // 'DynamicSegment'
```

This is used by `ast.toJSON(true)` to produce a human-readable version.

## Visualizing the AST

The `display()` method generates a formatted table:

```typescript
const ast = new AST('/users/:id.number?page.number#section')
console.log(ast.display())
```

The output includes ANSI colors to differentiate static segments, dynamic segments, types, and delimiters.
