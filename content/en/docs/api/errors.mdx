# Error Handling

url-ast reports parse errors in a structured way, without throwing exceptions. This allows you to inspect issues and decide how to handle them.

## `ErrorLog`

Each parse error is represented by an `ErrorLog` instance:

```typescript
import { ErrorLog } from 'url-ast'
```

The `ErrorLog` class stores information about errors found during parsing, including position and context.

---

## Checking for errors

### `hasErrors()`

Checks whether errors exist in the instance:

```typescript
const parsed = new Analyze('/users/[unclosed')

if (parsed.hasErrors()) {
  console.log('Parse errors found!')
}
```

### `displayErrors()`

Returns a formatted, human-readable string with details of all errors:

```typescript
const parsed = new Analyze('/users/[unclosed')

if (parsed.hasErrors()) {
  console.log(parsed.displayErrors())
  // Displays a detailed message about the unclosed bracket
}
```

---

## Accessing errors directly

Errors are available on the `errors` property of both `Analyze` and `AST` instances:

```typescript
const parsed = new Analyze('/invalid/[route')

// Via Analyze
console.log(parsed.errors)

// Via AST
console.log(parsed.ast.errors)
```

---

## Common error scenarios

| Scenario | Example | Description |
| :--- | :--- | :--- |
| Unclosed bracket | `/users/[id` | Missing `]` in dynamic segment |
| Malformed catch-all | `/docs/[..slug]` | Missing three dots (`...`) |
| Invalid type | `/:id.invalid` | Unrecognized type annotation |

---

## Error handling in production

url-ast doesn't throw exceptions — it **accumulates** errors and continues parsing as far as possible. This allows you to:

1. Always get a partial result, even with malformed URLs
2. Inspect errors and decide how to handle them
3. Display friendly error messages to the user

```typescript
import { Analyze } from 'url-ast'

function parseRoute(url: string, template: Analyze) {
  const result = new Analyze(url, template)

  if (result.hasErrors()) {
    console.warn('Problematic URL:', result.displayErrors())
    // Decide: return partial, fallback, or reject
  }

  return result.getParams()
}
```
