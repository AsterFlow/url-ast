# What is url-ast?

**url-ast** is a URL parser that transforms any URL into an **Abstract Syntax Tree (AST)**, allowing you to extract and manipulate each component — path, parameters, query string, fragment, and origin — in a structured and fully typed way.

## Why url-ast?

| Feature | `URLSearchParams` | `path-to-regexp` | **url-ast** |
| :--- | :---: | :---: | :---: |
| Path params parsing | ❌ | ✅ | ✅ |
| Query params parsing | ✅ (strings) | ❌ | ✅ (typed) |
| Fragment parsing | ❌ | ❌ | ✅ |
| Origin parsing (proto/host/port) | partial | ❌ | ✅ |
| **Automatic type casting** (`.number`, `.boolean`, `.array`, `.enum`) | ❌ | ❌ | ✅ |
| **TypeScript inference** from template literal | ❌ | ❌ | ✅ |
| Inspectable AST | ❌ | ❌ | ✅ |
| Binary serialization (`Buffer`) | ❌ | ❌ | ✅ |

## Core concept

The url-ast workflow revolves around two concepts:

1. **Template:** A string that describes the expected *shape* of the URL — which segments are dynamic, what types to expect, and which query params exist.
2. **Concrete URL:** The actual URL that will be parsed against the template.

<LiveCode initialCode={`import { Analyze } from 'url-ast'

// 1. Declares the template
const template = new Analyze('/users/:id.number?active.boolean')

// 2. Parses a real URL against the template
const result = new Analyze('/users/42?active=true', template)

result.getParams()        // { id: 42 }
result.getSearchParams()  // { active: true }`} />

The parser is **scannerless** — lexing and tree construction happen in a single pass, with no intermediate steps.

## What the AST offers

The generated AST is not just for data extraction. You can:

- **Inspect** the structure with `ast.toString()` (ANSI-colored formatted table)
- **Serialize** to JSON with `ast.toJSON()` or to binary with `ast.getBuffer()`
- **Restore** from a buffer with `AST.fromBuffer(buffer)`
- **Navigate** through nodes with `getNode()`, `getNodeByType()`, `getOriginNodes()`, `getPathNodes()`, etc.
