# Tratamento de Erros

O url-ast reporta **erros de parse** de forma estruturada: em vez de lançar exceções, ele os acumula silenciosamente na propriedade `errors`, para que você possa inspecionar problemas e decidir como lidar com eles. Recupere-os com `hasErrors()` / `formatErrors()`.

> **A conversão de valores é a exceção.** Erros de parse nunca lançam exceção, mas a **conversão de valores** (casting) lança. Ao chamar `getParams()` ou `getSearchParams()` em uma instância que possui um parser **base** e um valor falha na conversão (por exemplo, um valor não numérico para um parâmetro `.number`), o url-ast lança um `Error` comum cuja mensagem é o erro formatado.

## `ErrorLog`

Cada erro de parse é representado por uma instância de `ErrorLog`:

```typescript
import { ErrorLog } from 'url-ast'

new ErrorLog(code: string, message: string, start: number, end: number)
```

`ErrorLog` é um objeto de dados/formatação que armazena informações sobre um erro encontrado durante o parse, incluindo sua posição. Ele **não** é lançado como está — uma conversão que falha lança um `Error` comum (nunca um `ErrorLog`).

**Campos readonly**

| Campo | Tipo | Descrição |
| :--- | :--- | :--- |
| `code` | `string` | Um dos [códigos de erro](#códigos-de-erro) |
| `message` | `string` | Descrição legível do erro |
| `start` | `number` | Offset da coluna onde o erro começa |
| `end` | `number` | Offset da coluna onde o erro termina |

### `toString(input)`

```typescript
toString(input: string): string
```

Retorna uma string colorida no formato `Error [code] at col N: message`, seguida pelo `input` e um sublinhado com acento circunflexo (`^`) apontando para o trecho problemático.

## Verificando erros

### `hasErrors()`

Verifica se existem erros na instância:

<LiveCode initialCode={`const parsed = new Analyze('/users/[unclosed')

if (parsed.hasErrors()) {
  console.log('Encontrados erros de parse!')
}`} />

### `formatErrors()`

Retorna uma string formatada e legível com os detalhes de todos os erros:

<LiveCode initialCode={`const parsed = new Analyze('/users/[unclosed')

if (parsed.hasErrors()) {
  console.log(parsed.formatErrors())
  // Exibe uma mensagem detalhada sobre o colchete não fechado
}`} />

## Acessando erros diretamente

Os erros estão disponíveis na propriedade `errors` tanto da instância `Analyze` quanto da `AST`:

<LiveCode initialCode={`const parsed = new Analyze('/invalid/[route')

console.log(parsed.errors)

## Códigos de erro

Todo `ErrorLog` carrega um `code`. O url-ast define oito:

| Código | Significado |
| :--- | :--- |
| `E_INVALID_SYNTAX` | Sintaxe de URL/template malformada |
| `E_CONSECUTIVE_SLASHES` | Duas ou mais barras consecutivas no caminho |
| `E_DUPLICATE_PARAM` | O mesmo nome de parâmetro declarado duas vezes |
| `E_INVALID_CATCH_ALL` | Segmento catch-all malformado (`[...x]` / `[[...x]]`) |
| `E_DECODE_URI` | Percent-encoding inválido na URL (por exemplo, `%ZZ`) |
| `E_CAST_NUMBER` | O valor não pôde ser convertido para número |
| `E_CAST_BOOLEAN` | O valor não pôde ser convertido para booleano |
| `E_CAST_ENUM` | Valor fora das variantes de enum permitidas |

---

## Cenários comuns de erro

| Cenário | Exemplo | Código |
| :--- | :--- | :--- |
| Colchete não fechado | `/users/[id` | `E_INVALID_SYNTAX` |
| Catch-all malformado | `/docs/[..slug]` | `E_INVALID_CATCH_ALL` |
| Parâmetro duplicado | `/users/:id/posts/:id` | `E_DUPLICATE_PARAM` |

> Uma anotação de tipo desconhecida (por exemplo, `/:id.invalid`) **não** é um erro — o tipo silenciosamente recai para `string`.

## Tratamento de erros em produção

O url-ast não lança exceções em erros de **parse** — ele os **acumula** e continua o parse o máximo possível. (A conversão de valores em uma instância com parser base é a exceção: uma conversão que falha lança um `Error` comum.) Isso permite que você:

1. Sempre obtenha um resultado parcial, mesmo com URLs malformadas
2. Inspecione os erros e decida como tratá-los
3. Exiba mensagens de erro amigáveis para o usuário

```typescript
import { Analyze } from 'url-ast'

function parseRoute(url: string, template: Analyze) {
  const result = new Analyze(url, template)

  if (result.hasErrors()) {
    console.warn('URL com problemas:', result.formatErrors())
    // Decide: retornar parcial, fallback, ou rejeitar
  }

  return result.getParams()
}
```
